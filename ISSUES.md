## 🔴 P0 - 核心系统问题 ✅ 已完成

### Issue #1: 资源流转系统不符合规则

**严重程度**: 🔴 P0 | **影响范围**: 全局 | **状态**: ✅ 已完成 (v0.4)

**已实现的修改**:
- `GuestCard` 类型增加 `placedResources: Partial<Resources>` 字段
- `GameState` 新增 `pendingAllocation: Partial<Resources> | null` 待分配状态
- 行动区 1/2/6 获取 food/wine/coffee/cake 后进入待分配流程，不直接进资源池
- 新增 `allocatePendingResources` 函数：分配到厨房或指定客人卡
- 客人服务逻辑改为检查 `guest.placedResources` 而非 `player.resources`
- 厨房到客人卡的移动逻辑已适配
- 设置阶段邀请客人时初始化 `placedResources: {}`

## 🆕 P1.5 - 规则书核查问题（新发现）

> 以下问题源自 2026-06-12 导入的 MinerU 中文规则书，对照后发现当前实现与官方规则存在差异。

### Issue #15: 行动区1"食材市场"缺少蛋糕≤馅饼限制

**严重程度**: 🆕 P1.5 | **影响范围**: 行动区1 | **状态**: ✅ 已完成 (v0.6)

**规则要求**:
- 行动区1每有一个骰子，可以拿取1个**馅饼**或1个**蛋糕**
- 拿取的蛋糕数量**不能比馅饼多**（即 cake ≤ food）

**当前实现**:
- `performAreaAction1` 允许任意组合，没有蛋糕≤馅饼的限制

**需要修改的文件**:
- `src/game-logic/engine.ts` — `performAreaAction1`

**规则要求**:
- 行动区2每有一个骰子，可以拿取1个**红酒**或1个**咖啡**
- 拿取的咖啡数量**不能比红酒多**（即 coffee ≤ wine）

### Issue #17: 每回合缺少"邀请客人"可选行动

**严重程度**: 🆕 P1.5 | **影响范围**: 回合流程 | **状态**: ✅ 已完成 (v0.6)

**规则要求**:
- 每回合流程：1) **邀请客人（可选行动）** → 2) 拿取骰子执行行动（必选行动）→ 额外行动（可选）
- 邀请客人是从游戏版图上5张正面朝上的房客中拿取1张，支付费用放到咖啡厅空桌
- 咖啡厅3张餐桌都被占据则不能拿取

**当前实现**:
- 邀请客人只能通过特定行动（如行动区5后续操作），不是每回合的标准可选行动
- 缺少从游戏版图公共区邀请客人的流程

**需要修改的文件**:
- `src/game-logic/engine.ts` — 添加每回合邀请客人的可选步骤
- `src/components/ActionPanel.tsx` — 添加"邀请客人"按钮作为行动前可选操作
- `src/store/gameStore.ts` — 添加对应 store action

---

### Issue #18: 轮结束缺少顺位板块顺时针传递

**严重程度**: 🆕 P1.5 | **影响范围**: 轮次切换 | **状态**: ✅ 已完成 (v0.6)

**规则要求**:
- 每轮结束时，把顺位板块按**顺时针**方向传给下一个玩家

**需要修改的文件**:

### Issue #19: 资源中文名称需统一（"馅饼" vs "面包"）

**严重程度**: 🆕 P1.5 | **影响范围**: UI / 数据 | **状态**: ✅ 已完成 (v0.6)

**规则要求**:
- 中文规则书使用的资源名称为：**馅饼**（非"面包"）、**红酒**、**咖啡**、**蛋糕**、**金钱**

**当前实现**:
- 代码和数据中混合使用"food/面包"和"馅饼"，UI 显示为"面包"
- `RES_NAMES_CN` 中使用了"面包"而非"馅饼"

**需要修改的文件**:
- `src/components/SetupPhase.tsx` — `RES_NAMES_CN` 中 `food` 改为 `馅饼`
- `src/components/ActionPanel.tsx` — 如有资源名称映射也需同步
- `src/data/guests.ts` — 客人需求描述中使用"馅饼"

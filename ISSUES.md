## 🔴 P0 - 核心系统问题

### Issue #1: 资源流转系统不符合规则

**严重程度**: 🔴 P0 | **影响范围**: 全局 | **状态**: 待修复

**规则要求**:
- 获得食材/酒水后，必须**立即**将其放到客人卡片的"需求区域"上，或放到厨房中
- 客人服务时，从客人卡片上已放置的资源来判断需求是否满足
- 厨房是资源暂存区，通过额外行动"移动厨房"将资源移到客人卡片上

**当前实现**:
- 食材/酒水直接进入 `player.resources` 抽象资源池
- 客人服务从 `player.resources` 扣除
- 厨房 (`player.kitchen`) 只是一个独立的数据结构，没有承担规则中的中枢角色

**需要修改的文件**:
- `src/game-logic/engine.ts` — 所有资源获取/消费逻辑
- `src/store/gameStore.ts` — 所有 action 的 dispatch
- `src/types/game.ts` — 可能需要调整 `Resources` 和 `GuestCard` 类型（客人卡片上需要追踪已放置的资源）
- 所有与资源获取相关的 UI 组件

**修改思路**:
1. `GuestCard` 增加 `placedResources: Partial<Resources>` 字段，追踪已放到客人卡上的资源
2. 行动区 1/2/4/6 获取资源后，弹出 UI 让玩家分配到厨房或指定客人卡
3. 客人服务判定改为：检查客人卡片上 `placedResources` 是否满足 `requirements`
4. 厨房资源通过额外行动"移动厨房"转移到客人卡片上

### Issue #2: 员工卡轮选机制错误

**严重程度**: 🔴 P0 | **影响范围**: 设置阶段 | **状态**: 待修复

**规则要求**:
- 每人获得 6 张员工卡（暗牌），选 1 张保留，剩下的传给**左手边玩家**（顺时针）
- 重复此流程，直到每人选满 6 张

**当前实现**:
- 每人从自己的 6 张中逐个挑选（蛇形顺序），没有传牌机制

**需要修改的文件**:
- `src/game-logic/engine.ts` — `drawStaffCardsForPlayer` / `pickStaffCardForDraft`
- `src/store/gameStore.ts` — `drawStaffCards` / `pickStaffCard`
- `src/types/game.ts` — `Player` 类型可能需要 `passingHand` 字段

**修改思路**:
1. 每人从自己的 `draftHand` 中选 1 张
2. 选完后，将剩余的牌传给左手玩家（build a new draftHand for each player）
3. 一人选完后判断是否所有人都已选满 6 张，未满则继续轮次

---

### Issue #3: 骰子遮盖顺位机制未实现

**严重程度**: 🔴 P0 | **影响范围**: 行动流程 | **状态**: 待修复

**规则要求**:
- 执行行动时，从行动区取走的**那颗骰子**用来遮住自己顺位板上**当前最小的可见数字**
- 两个数字都被遮住后，该玩家本轮不能再行动
- 「跳过」也是遮盖当前最小数字

**当前实现**:
- 使用 `actionsPerformed` 计数器来跟踪行动次数（0/1/2）
- 没有骰子遮盖顺位板的物理概念

**需要修改的文件**:
- `src/game-logic/engine.ts` — `performTurnAction` / `skipTurn` 中的遮盖逻辑
- `src/types/game.ts` — `TurnOrderCovered` 类型和遮盖逻辑
- `src/store/gameStore.ts` — 相关 action

**修改思路**:
1. 玩家执行行动后，遮盖顺位板上"当前最小可见数字"（而非第一个/第二个）
2. `TurnOrderCovered` 改为基于顺位板数字的可见性，而不是 first/second
3. 跳过行动也遮盖当前最小可见数字

## 🟠 P1 - 玩法正确性问题

### Issue #4: 额外行动"使用员工能力"应为选1张

**严重程度**: 🟠 P1 | **影响范围**: 额外行动 | **状态**: 待修复

**规则要求**:
- 使用顺位板上的"员工能力"额外行动时，只能**选择 1 张** `once_per_round` 的员工卡使用

**当前实现**:
- `performExtraActionUseStaffAbility` 一次性触发**所有** `once_per_round` 员工卡

**需要修改的文件**:
- `src/store/gameStore.ts` — `performExtraActionUseStaffAbility`
- `src/components/ActionPanel.tsx` 或相关 UI — 需要让用户选择具体使用哪张卡

### Issue #7: 入住客人必须占据特定颜色房间

**严重程度**: 🟠 P1 | **影响范围**: 客人入住 | **状态**: 已部分实现

**规则要求**:
- 红色客人 → 红色房间，黄色客人 → 黄色房间，蓝色客人 → 蓝色房间

**当前实现**:
- `canServeGuest` 中已有颜色匹配逻辑
- 但 `serveGuest` (旧版) 中缺少颜色匹配检查
- 需要确认 UI 是否正确提示可选房间

**需要修改的文件**:
- `src/game-logic/engine.ts` — 确认所有 serve 路径都检查颜色
- `src/components/` — 客人选房 UI

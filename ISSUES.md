## 🔴 P0 - 核心系统问题 ✅ 已完成

### Issue #1: 资源流转系统不符合规则

**严重程度**: 🔴 P0 | **影响范围**: 全局 | **状态**: ✅ 已完成 (v0.4)

**已实现的修改**:
- `GuestCard` 类型增加 `placedResources: Partial<Resources>` 字段
- `GameState` 新增 `pendingAllocation: Partial<Resources> | null` 待分配状态
- 行动区 1/2/6 获取 food/wine/coffee/cake 后进入待分配流程，不直接进资源池
- 新增 `allocatePendingResources` 函数：分配到厨房或指定客人卡
- 客人服务逻辑改为检查 `guest.placedResources` 而非 `player.resources`
- 厨房到客人卡的移动逻辑已适配
- 设置阶段邀请客人时初始化 `placedResources: {}`

---

### Issue #2: 员工卡轮选机制错误

**严重程度**: 🔴 P0 | **影响范围**: 设置阶段 | **状态**: ✅ 已完成 (v0.4)

**已实现的修改**:
- `Player` 类型新增 `passingHand: StaffCard[]` 字段
- `GameState` 新增 `setupPassingDirection` 和 `setupStaffRound` 字段
- `pickStaffCardForDraft` 实现真正的轮抽：选1张保留 → 剩余牌传给左手玩家 → 逐轮重复
- 每轮所有玩家各选1张，6轮后每人拥有6张员工卡

---

### Issue #3: 骰子遮盖顺位机制未实现

**严重程度**: 🔴 P0 | **影响范围**: 行动流程 | **状态**: ✅ 已完成 (v0.4)

**已实现的修改**:
- 移除 `TurnOrderCovered` 类型和 `turnOrderCovered` 字段
- 移除 `actionsPerformed` 计数器，替换为 `coveredSlots: number` (0/1/2)
- `getNextActionPlayer` 基于 `coveredSlots` 判断玩家是否可行动
- 跳过回合机制保留 `hasPassedInCycle`，加入 `coveredSlots` 的联合判断
- `startNextRound` 重置 `coveredSlots: 0`
- UI 组件统一更新

## 🟠 P1 - 玩法正确性问题

### Issue #4: 额外行动"使用员工能力"应为选1张

**严重程度**: 🟠 P1 | **影响范围**: 额外行动 | **状态**: 待修复

**规则要求**:
- 使用顺位板上的"员工能力"额外行动时，只能**选择 1 张** `once_per_round` 的员工卡使用

**当前实现**:
- `performExtraActionUseStaffAbility` 一次性触发**所有** `once_per_round` 员工卡

**需要修改的文件**:
- `src/store/gameStore.ts` — `performExtraActionUseStaffAbility`
- `src/components/ActionPanel.tsx` 或相关 UI — 需要让用户选择具体使用哪张卡

---

### Issue #5: 设置阶段客人邀请顺序有误

**严重程度**: 🟠 P1 | **影响范围**: 设置阶段 | **状态**: 待修复

**规则要求**:
- 从顺位板数字最小的玩家开始，**逆时针**每人免费邀请 1 位客人

**当前实现**:
- `getSetupOrder` 函数计算顺序为从起始玩家逆时针，但需要确认是否与规则完全一致
- 需要验证：起始玩家 → 他的右手玩家 → ... 还是其他顺序

**需要修改的文件**:
- `src/game-logic/engine.ts` — `getSetupOrder` / `pickSetupGuest`

> **说明**: 资源获取后立即分配已在 Issue #1 中实现，原 Issue #6 取消。

### Issue #7: 入住客人必须占据特定颜色房间

**严重程度**: 🟠 P1 | **影响范围**: 客人入住 | **状态**: 已部分实现

**规则要求**:
- 红色客人 → 红色房间，黄色客人 → 黄色房间，蓝色客人 → 蓝色房间
- 绿色客人 → 任意颜色房间

**当前实现**:
- `canServeGuest` 中已有颜色匹配逻辑
- 但 `serveGuest` (旧版) 中缺少颜色匹配检查
- 需要确认 UI 是否正确提示可选房间

**需要修改的文件**:
- `src/game-logic/engine.ts` — 确认所有 serve 路径都检查颜色
- `src/components/` — 客人选房 UI
